1. Актуальные алгоритмы классификации

В современных задачах классификации чаще всего используют несколько групп моделей.

Классические ML-алгоритмы:

Группа	Алгоритмы	Где применяются
Линейные модели	Logistic Regression, SGD Classifier	быстрые бейзлайны, текст, табличные данные
Метрические методы	k-Nearest Neighbors	небольшие датасеты, простые задачи
Вероятностные модели	Naive Bayes	текстовая классификация, спам-фильтры
Деревья решений	Decision Tree	интерпретируемые модели
Ансамбли деревьев	Random Forest, Extra Trees	табличные данные, устойчивые бейзлайны
Градиентный бустинг	XGBoost, LightGBM, CatBoost	один из сильнейших вариантов для табличных данных
Метод опорных векторов	SVM / SVC	малые и средние датасеты, сложные границы классов
Нейронные сети	MLP, CNN, RNN, Transformers	изображения, текст, аудио, сложные данные

В документации scikit-learn классификация описывается как задача отнесения объекта к одной из категорий; среди стандартных методов есть SVM, nearest neighbors, decision trees, random forests, gradient boosting, logistic regression и другие.  ￼

Для текста часто используют Naive Bayes, Logistic Regression, SVM, SGD Classifier, а в более сложных задачах — Transformer-модели.

4. Интеграция обученной модели с веб-сервисом

Общая схема интеграции ML-модели во Flask / Django / FastAPI:

1. Подготовить данные
    Очистить данные, обработать пропуски, закодировать категориальные признаки, нормализовать числовые признаки при необходимости.
2. Обучить модель
    Например, обучить RandomForestClassifier, CatBoostClassifier, XGBClassifier или другую модель.
3. Оценить качество
    Посчитать Accuracy, Precision, Recall, F1-score, ROC-AUC, confusion matrix.
4. Сохранить модель
    Обычно используют pickle или joblib. В документации scikit-learn указано, что joblib часто эффективнее для больших моделей и массивов NumPy.  ￼
5. Создать веб-приложение
    Например, на FastAPI создать endpoint /predict, который принимает JSON с признаками объекта.
6. Загрузить модель при запуске сервера
    Модель загружается один раз из файла, чтобы не обучать её заново при каждом запросе.
7. Обработать входной запрос
    Получить данные пользователя, привести их к тому же формату, который использовался при обучении.
8. Сделать предсказание
    Передать признаки в модель: model.predict(...) или model.predict_proba(...).
9. Вернуть результат клиенту
    Например: класс, вероятность класса, статус ответа.
10. Развернуть сервис
    Запустить через uvicorn для FastAPI или через gunicorn/uwsgi для Flask/Django. Можно завернуть приложение в Docker и разместить на сервере.

5. Пример логики FastAPI

```python
from fastapi import FastAPI
from pydantic import BaseModel
import joblib
import numpy as np
app = FastAPI()
model = joblib.load("model.pkl")
class InputData(BaseModel):
    age: float
    income: float
    score: float
@app.post("/predict")
def predict(data: InputData):
    features = np.array([[data.age, data.income, data.score]])
    prediction = model.predict(features)[0]
    probability = model.predict_proba(features).max()
    return {
        "class": int(prediction),
        "probability": float(probability)
    }
```
FastAPI часто используют для ML-сервисов, потому что он быстрый, удобный для REST API и автоматически создаёт документацию Swagger. Современные туториалы по деплою scikit-learn моделей через FastAPI используют именно схему: обучение модели → сохранение → загрузка в API → endpoint для предсказания.  ￼
